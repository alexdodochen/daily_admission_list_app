============================================================
  行政總醫師．排班．Key班．入院  使用方法
============================================================

這是心臟內科行政總醫師的工作自動化工具，三大功能：
  ① 排班   ② Key 班   ③ 每日入院清單（含 EMR / 導管排程 / LINE 推播）


------------------------------------------------------------
【第一步】安裝（不用安裝，解壓即用）
------------------------------------------------------------
  1. 把 admission-app.zip 解壓到任何資料夾
     （例如桌面、D 槽，路徑不要有特殊符號即可）
  2. 進入解壓出來的「行政總醫師.排班.Key班.入院」資料夾
  3. 雙擊「行政總醫師.排班.Key班.入院.exe」
  4. 會自動打開瀏覽器到 http://127.0.0.1:8766
     （第一次啟動稍慢，請等 5~10 秒；黑色命令視窗不要關掉，
       那是程式本體，關掉 = 關掉系統）

  ※ 若雙擊時跳出「Windows 已保護你的電腦」藍色視窗：
     按「其他資訊」→「仍要執行」即可（這支程式未做數位簽章，
     不是病毒，是正常現象）。


------------------------------------------------------------
【第二步】第一次設定 — 放入我另外給你的 4 個檔案
------------------------------------------------------------
我會「另外私下」傳給你 4 樣東西：
  (A) service_account.json          ← Google 憑證，1 個檔
  (B) cathlab_id_maps.json   \
  (C) doctor_codes.json       )     ← 導管排程設定，共 3 個檔
  (D) cathlab_schedule.json  /

這些檔案「不在」壓縮檔裡（基於病人隱私，公開檔不附帶），
要你自己放進「設定資料夾」一次，之後永久有效、更新也不會弄丟。

設定資料夾路徑（直接複製貼到檔案總管網址列）：

    %LOCALAPPDATA%\admission-app

  也就是： C:\Users\你的Windows帳號\AppData\Local\admission-app

放法（很簡單，4 個檔全部丟進「同一個」資料夾就好）：
  1. 開啟上面那個資料夾（admission-app）
  2. 把我給你的 4 個檔案（A 的 service_account.json
     + B C D 三個 cathlab JSON）「全部直接丟進」admission-app
     資料夾裡。不用新建子資料夾、不用改檔名。
     （service_account.json 就算還是 Google 下載時的原始長檔名
      也沒關係，程式會自己認）

  完成後結構長這樣：

    admission-app\
      ├─ service_account.json          ← (A)
      ├─ cathlab_id_maps.json          ← (B)
      ├─ doctor_codes.json             ← (C)
      ├─ cathlab_schedule.json         ← (D)
      └─ config.json                   （程式自動產生，不用理）

  3. 回到瀏覽器設定頁，按「測試連線」按鈕
     看到 Sheet / 服務帳號 顯示綠色成功就完成了
     （不用重開程式；若還沒成功，再按一次測試連線）


------------------------------------------------------------
【第三步】填你自己的帳密（在設定頁）
------------------------------------------------------------
  - LLM API 金鑰（圖片辨識用，設定頁有申請教學連結）
  - WEBCVIS 帳號 / 密碼（導管排程 key-in 用）
  - LINE Bot Token / 群組 ID（入院名單推播用，設定頁有建立教學）

  填完按綠色「儲存」。之後就能正常使用三大功能。


------------------------------------------------------------
【更新】
------------------------------------------------------------
  程式內按「更新」按鈕 → 自動下載最新版、換檔、重開。
  你放在設定資料夾(admission-app)裡的憑證與 cathlab 檔
  「不會」被更新覆蓋，免重設。

  ※ 第一次例外：如果你目前這版是「更新鈕修好之前」的舊版，
    更新鈕會出現「只支援 git checkout」之類錯誤 —— 這版的
    更新功能本身有 bug。請「手動」到下面網址下載一次最新 zip、
    解壓覆蓋使用（設定資料夾不受影響）：
      https://github.com/alexdodochen/daily_admission_list_app/releases/latest
    換成新版後，以後按「更新」鈕就會正常，不必再手動。


------------------------------------------------------------
【回報問題給開發者】
------------------------------------------------------------
  出狀況時，按頂端的「🐞 回報問題」：
    1. 填「哪一步出問題 / 描述 / 錯誤訊息」（最近的紅色錯誤
       會自動帶入）。
    2. 按「產生回報內容」——系統會自動隱藏病歷號 / 姓名 /
       金鑰 / 密碼，並附上版本與最近 log。
    3. 二選一：
       ①「開 GitHub 回報」= 公開 issue（送出前請自己再看一眼
          預覽，確認沒有病人資料）。
       ②「存成檔案」= 存到 admission-app\bug_reports\，再用
          LINE / email 私下把那個檔傳給陳常胤醫師。
  （這個功能不會自動連網送出，一定是你按了才送。）


------------------------------------------------------------
【常見問題】
------------------------------------------------------------
Q1. 雙擊 exe 沒反應 / 一閃就關 / 說連接埠被佔用
   → 多半是「已經開著一個了」。先到瀏覽器開
     http://127.0.0.1:8766 看看是不是其實已經在跑。
     若要重開：工作管理員 → 結束所有
     「行政總醫師.排班.Key班.入院.exe」→ 再雙擊一次。

Q2. 設定頁一直顯示找不到 Sheet / 服務帳號
   → 檢查 service_account.json 有沒有丟進 admission-app 資料夾。
     放完按設定頁「測試連線」（不用重開程式）。

Q3. 導管排程那一步出現「找不到導管設定檔」
   → cathlab 的 3 個 JSON 沒放對。三個都要丟進
     admission-app 資料夾（跟 service_account.json 同一層，
     不用建子資料夾）。放完按設定頁「測試連線」即可。

Q4. 黑色命令視窗可以關嗎？
   → 不行。那是程式本體，關掉等於關掉整個系統。
     用完要關閉系統時，才關那個黑視窗。
