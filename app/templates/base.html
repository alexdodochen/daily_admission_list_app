📋 查閱 Google Sheet

先選 Sheet 來源，再選分頁開始查閱（可直接編輯儲存格）。

📖 操作說明

這個 app 把每天的住院 + 隔天的心導管排程自動化。資料全部存在你自己的 Google Sheet，所有 API key 都在本機。

第一次使用

  1. 設定頁 填：LLM key (Anthropic / OpenAI / Gemini 任一) → Google service account JSON 路徑 → Sheet ID → WEBCVIS 帳密 → LINE token。「測試連線」要兩個 ✓ 才能繼續。
  2. 把 service account 的 email 加入該 Sheet 的「編輯者」（或把 Sheet 設「知道連結都可以編輯」）。

每天 6 步

  1. Step 1 OCR — 拖入院名單截圖 → LLM 辨識成主表 → 對病歷號 → 寫入 Sheet（OCR 會自動填日期，diff 會更新子表格）。
  2. Step 2 生成 subtable — 依主表順序，把病人按主治醫師分組，產生子表格。不抽籤、不寫入院序。已有子表格會拒絕覆寫。
  3. Step 3 EMR 擷取 — 自己登入 EMR 後貼 session URL → Playwright 抓 SOAP → 自動偵測 術前診斷 / 預計心導管（含 plan-section 規則）→ 寫回子表格的 EMR / 術前診斷 / 預計心導管 + 修正主表 姓名 / 性別 / 年齡。診斷 / 心導管 在 EMR 卡片可即時編輯（▼ 下拉或自填）。
  4. Step 4 抽籤 + 入院序 — 三層 pin（同醫師排序欄 / 病人 / 醫師）→「② 首次抽籤」整段重寫入院序（會清掉 備註(住服) / 改期）；改完 術前診斷 / 預計心導管 / 同醫師排序 之後用「③ 整合寫入」（保留 備註(住服) / 改期）。
  5. Step 5 導管 keyin — Plan (dry-run) / Verify (查 WEBCVIS) / Keyin (ADD + UPT)。術前診斷 / 預計心導管 自填的內容會走 OTHERS_PDI / 備註。
  6. Step 6 LINE 推播 — 讀 入院序 / 主治 / 姓名 / 備註(住服) 四欄 → 預覽訊息 → 推到指定 group。

遇到問題

  • 頂端紅燈 = 設定還沒填好 → 回設定頁。
  • Step 3 / 5 卡住 = EMR / WEBCVIS session 過期 → 重新登入再貼一次 URL。
  • OCR 病歷號錯 = Sheet 主資料 I 欄直接改後重跑 Step 3 (EMR)。EMR 會用 #divUserSpec 反查校正姓名/年齡/性別。
  • 所有設定存在 %LOCALAPPDATA%\admission-app\config.json，更新 app 不會被覆蓋。

月份值班排班自動產生器。寫入 schedule_sheet_id 試算表的 YYYYMM 工作表 + 累積統計。

5 步流程

  1. ① 月份選擇 — 點月份 chip（預設今天的月份），系統載入該月日曆 + 你想排的醫師。
  2. ② 設定 baseline — 在表格內手填已知排班（例：某天某醫師確定值班）。空格留給 solver 填。
  3. ③ Compute targets — 系統依當月 平日/週五/週六/週日/假日 數 + 你的累積統計，算出每位醫師該月的目標班數。
  4. ④ Solve — backtracking solver 嘗試分派班次，符合 QOD（不能連兩天）規則。失敗會自動 relax。
  5. ⑤ Write — 把結果寫到 schedule_sheet 的 YYYYMM 工作表 + 更新 值班總數統計。

注意

  • 需要在 設定頁 填入 schedule_sheet_id（與入院 Sheet 可不同），且 service account 是該 Sheet 編輯者。
  • 累積統計表（值班總數統計）必須先存在；首次使用要手動建立含醫師欄位 + 5 個分類欄。
  • solver 偶爾會跑分鐘級時間（uniform-zero baseline 特別久）— 沒回應就再等。

把排班 Excel 匯入後，自動 keyin 到 EDR 系統的班表。本卡片需要 EDR 帳密 + 至少一個 Excel 檔。

3 步流程

  1. ① 上傳 Excel — 拖入該月已填好的排班 Excel（.xlsx 或 .xls）。系統會解析並顯示預覽，可確認班次正確。
  2. ② 填 EDR 帳密 + 開始 — 填一次帳密就開始，Playwright 會開瀏覽器登入 EDR、找到對應月份、把每個班次塞進去。
  3. ③ 進度條 / log — 即時 WebSocket 顯示進度。卡住可按「取消」。下一次重跑會跳過已 keyin 成功的班次。

注意

  • EDR 介面變動時 keyin 會 fail — 先單機手動 key 一次確認 DOM 結構，再回報給 Claude 修正 selector。
  • Excel 必須是排班程式產生的格式，自己手畫的會被解析錯。
  • 失敗的班次會記在 log，可以下載 csv 自己補。

🐞 回報問題給開發者

系統會自動附上版本、環境、最近的錯誤與 log，並 自動隱藏病歷號 / 姓名 / 金鑰 / 密碼。送出前你可以再看一次內容。
